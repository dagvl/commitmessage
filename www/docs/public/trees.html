Module Hierarchy

  • commitmessage: A MVC framework for capturing information from scm systems.
    • attribute
    • controllers: Controllers parse information from a scm into the Model and then execute the Views.
      • cvs: The controller and utils for the CVS SCM (http://www.cvshome.org).
      • svn: The controller and utils for the Subversion SCM (http://subversion.tigris.org).
    • exceptions: Contains the CmException class.
    • Itpl: String interpolation for Python (by Ka-Ping Yee, 14 Feb 2000).
    • main: Bootstraps the commitmessage framework by reading in the configuration file and initializing the controller.
    • model: Provides the model classes for the commitmessage framework.
    • util: Provides some basic utility classes, such as wrapping the ConfigParser to add commitmessage-specific logic to the view/module lookups.
    • views: Views work against the object model to send out information to outside systems.
      • email: Provides several email Views for the commitmessage framework.
      • misc: Provides several email Views for the commitmessage framework.
      • post: Provides hooks into various systems via HTTP post.

Class Hierarchy

  • commitmessage.Itpl.Itpl: Class representing a string with interpolation abilities.
  • commitmessage.Itpl.ItplFile: A file object that filters each write() through an interpolator.
  • commitmessage.model.Controller: A base implementation for SCM-specific controllers to extend; mostly does the generic Views handling.
    • commitmessage.controllers.cvs.CvsController: Translates CVS loginfo/commitinfo information into the Model.
    • commitmessage.controllers.svn.SvnController: Uses 'svnlook' to pull data from svn repositories.
  • commitmessage.model.Directory: Represents a directory that has been affected by the commit.
  • commitmessage.model.File: Represents a file that has been affected by the commit.
  • commitmessage.model.Model: Wraps the Files and Directorys that get sent between the Controller and Views.
  • commitmessage.model.View: Provides a base View for specific implementations to extend.
    • commitmessage.views.email.BaseEmailView: Provides a basic implementation of sending an email for other style-specific email views to extend.
      • commitmessage.views.email.HtmlEmailView: Sends out a HTML formatted email with the commit info in tables.
      • commitmessage.views.email.TigrisStyleEmailView: Sends out an email formatted in a style mimicking Tigris commit emails.
    • commitmessage.views.misc.DumpView: Provides unit testing capabilities by doing a simple dump of the Model to a text file.
    • commitmessage.views.post.PostView
  • exceptions.Exception: Common base class for all exceptions.
    • commitmessage.exceptions.CmException: The exception for all commitmessage-related exceptions.
    • exceptions.StandardError: Base class for all standard Python exceptions.
      • exceptions.ValueError: Inappropriate argument value (of correct type).
        • commitmessage.Itpl.ItplError
