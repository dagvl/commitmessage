Mission

commitmessage provides a modular, easy-to-use approach to sending CVS commit information to other systems.

Many scripts already exist to send out email diffs for commits, post information into bug tracking systems, etc., but they all have tight coupling between the logic of collecting information from CVS and the logic of sending relevant information out to the other system.

commitmessage takes smarter approach: MVC.

Two scripts (commitmessage_loginfo.pl and commitmessage_commitinfo.pl) act the Controllers for the process and build a Model of the entire commit (not just per-directory) that can then be passed on to various Views.

The Views are written as Perl packages to allow the Controllers to interact with the Views as objects.

Setting up different Views only requires editing a very simple property file: commitmessage.config. The Controllers never need editing.

Also, the commitmessage.config can contain various properties on either a global or per-module basis to allow a single View package to be reused by various modules.

The overall affect is cutting down on the extreme duplication between scripts that used to occur just to change the target system or even simple properties such as Urls or mailing address for different modules.

Related resources
